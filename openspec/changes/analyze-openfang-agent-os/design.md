### 1.2 核心价值主张

OpenFang 定位为 **"Agent Operating System"** — 不是聊天机器人框架，不是 Python 包装器，而是一个完整的自主代理操作系统。

**核心差异化**：
- **真正的自主运行**：传统 Agent 框架等待用户输入，OpenFang 的 Hands 可以按调度计划 24/7 自主运行
- **单二进制部署**：整个系统编译为 ~32MB 单一可执行文件，无需 pip install、Docker pull
- **生产级安全**：16 层独立安全系统，而非事后添加的安全层
- **Rust 原生架构**：从零开始构建，性能和安全性优于 Python/TypeScript 框架

### 2.2 核心技术栈

| 组件 | 技术选型 | 设计理由 |
|------|----------|----------|
| **运行时** | Rust + Tokio | 零成本抽象、内存安全、高并发 |
| **沙箱** | WASM + Fuel Metering | 双重计量（燃料+epoch），防止失控代码 |
| **存储** | SQLite + 向量扩展 | 嵌入式、零配置、支持向量搜索 |
| **审计** | Merkle Hash Chain | 密码学链接，防篡改 |
| **网络** | OFP P2P Protocol | HMAC-SHA256 互认证 |
| **桌面** | Tauri 2.0 | Rust原生、轻量级、跨平台 |
| **前端** | HTML/CSS/JS (5.3%) | 仪表板界面 |

### 2.3 16层安全系统

OpenFang 的安全设计采用"纵深防御"策略，每层独立可测试：

| # | 安全系统 | 功能描述 |
|---|----------|----------|
| 1 | **WASM双计量沙箱** | 工具代码在WebAssembly中运行，燃料计量+epoch中断，看门狗线程杀死失控代码 |
| 2 | **Merkle哈希链审计** | 每个操作密码学链接到前一个，篡改一个条目整个链断裂 |
| 3 | **信息流污点追踪** | 标签通过执行传播，从源头到汇点追踪秘密 |
| 4 | **Ed25519签名Agent清单** | 每个Agent身份和能力集密码学签名 |
| 5 | **SSRF防护** | 阻止私有IP、云元数据端点、DNS重绑定攻击 |
| 6 | **秘密零化** | `Zeroizing<String>` 自动从内存擦除API密钥 |
| 7 | **OFP互认证** | HMAC-SHA256基于nonce、常量时间验证 |
| 8 | **能力门控** | RBAC，Agent声明所需工具，内核强制执行 |
| 9 | **安全头** | CSP、X-Frame-Options、HSTS、X-Content-Type-Options |
| 10 | **健康端点脱敏** | 公共健康检查返回最小信息 |
| 11 | **子进程沙箱** | `env_clear()` + 选择性变量传递，进程树隔离 |
| 12 | **提示注入扫描器** | 检测覆盖尝试、数据泄露模式、shell引用注入 |
| 13 | **循环防护** | SHA256工具调用循环检测，熔断器 |
| 14 | **会话修复** | 7阶段消息历史验证，自动恢复 |
| 15 | **路径遍历防护** | 规范化+符号链接逃逸防护 |
| 16 | **GCRA限流器** | 成本感知令牌桶，每IP跟踪 |

## 3. Hands 自主代理系统

### 3.1 设计理念

**核心创新**：Hands 不是等待用户输入的聊天机器人，而是按调度计划自主运行的能力包。

每个 Hand 包含：
- **HAND.toml** — 清单文件，声明工具、设置、要求、仪表板指标
- **System Prompt** — 多阶段操作手册（500+ 词的专业流程）
- **SKILL.md** — 运行时注入的领域专业知识
- **Guardrails** — 敏感操作的审批门（如购买前需确认）

### 3.2 7个内置Hands

| Hand | 功能描述 | 技术实现 |
|------|----------|----------|
| **Clip** | YouTube视频剪辑，自动识别精彩片段，生成竖屏短视频，添加字幕和AI配音，发布到Telegram/WhatsApp | 8阶段流水线：yt-dlp下载 → 5种STT后端 → FFmpeg处理 → 发布 |
| **Lead** | 每日运行，发现匹配ICP的潜在客户，网络研究丰富信息，0-100评分，去重，交付CSV/JSON/Markdown | ICP建模 → 网络爬取 → 评分算法 → 去重 → 格式化输出 |
| **Collector** | OSINT级情报收集，持续监控目标（公司/人物/主题），变化检测、情感追踪、知识图谱构建、关键警报 | 多源监控 → 变化检测 → 情感分析 → 图谱构建 |
| **Predictor** | 超级预测引擎，多源信号收集，构建校准推理链，带置信区间的预测，Brier评分跟踪准确率 | 信号聚合 → 推理链 → 置信校准 → Brier评分 |
| **Researcher** | 深度自主研究，多源交叉引用，CRAAP标准评估可信度，APA格式引用报告，多语言支持 | CRAAP评估 → 交叉验证 → APA格式化 |
| **Twitter** | 自主Twitter/X账户管理，7种内容格式轮换，最佳时间调度，提及回复，性能追踪，审批队列 | 内容生成 → 调度优化 → 审批队列 → 分析 |
| **Browser** | Web自动化代理，站点导航、表单填写、按钮点击、多步骤工作流，Playwright桥接，会话持久化，**强制购买审批门** | Playwright → 会话管理 → 审批门控 |

### 3.3 Hands vs 传统Agent对比

| 维度 | 传统Agent框架 | OpenFang Hands |
|------|---------------|----------------|
| **运行模式** | 被动等待用户输入 | 主动按计划运行 |
| **调度能力** | 需要外部调度器 | 内置调度系统 |
| **状态管理** | 会话级 | 持久化 + 恢复 |
| **安全控制** | 后置添加 | 设计阶段内置 |
| **部署复杂度** | pip install + 依赖管理 | 单一二进制 |

### 7.1 优势

| 方面 | 详情 |
|------|------|
| **性能** | Rust原生，冷启动180ms，内存40MB，远超Python框架 |
| **安全** | 16层纵深防御，WASM沙箱，Merkle审计链 |

### 7.2 风险

| 风险类型 | 详情 | 缓解措施 |
|----------|------|----------|
| **成熟度** | v0.2.7，首个公开版本 | 等待v1.0，或锁定特定commit |
| **社区** | 单一贡献者(jaberjaber23) | 关注社区发展，考虑参与贡献 |
| **文档** | 官网文档较新，可能不完整 | 查阅源码和GitHub Issues |
| **稳定性** | GitHub Issues显示部分功能问题 | 测试验证关键功能 |
| **学习曲线** | Rust + 新概念 | 评估团队技术栈适配 |

### 7.3 已知问题

根据 GitHub Issues 分析：
- 部分本地模型集成存在问题
- 错误信息有时不够友好
- 早期版本的API格式问题已在后续版本修复

- 需要稳定API的生产系统（仍在快速迭代）
